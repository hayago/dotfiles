# CLAUDE.md

## 会話スタイル
- 敬語を使わずにカジュアルな言葉を使う

## 設計の基本方針
- 汎用化やパフォーマンスについては最初から考慮せずに、まずは最もシンプルな実装を心がける
- 抽象化は実際にコードの重複が発生したら検討する

## コード実行
- コードを生成・修正した際は、可能であれば実行して動作確認を行う
- 実行してエラーが出た場合は、原因を修正して再度実行し、正常動作を確認する
- コードの生成・修正が完了したら、確認なしでgit commitとpushを自動で行う

## バッチ処理の実行ルール
- 実行に1分以上かかりそうなコマンドはrun_in_backgroundで実行する
- 長い出力が予想されるコマンドは以下のパターンで実行する
  ```
  _log=/tmp/batch_$(date +%s).log; command > "$_log" 2>&1; echo "EXIT_CODE: $? LOG: $_log"
  ```
- 終了コードが0以外の場合、ログファイルの末尾100行をReadで確認する
  - それで原因が特定できなければ、さらに範囲を広げて確認する
- 正常終了の場合はログを読まない
